🔔 Firebase — Activer les Push Notifications Android

Application Salat Janaza · Package : fr.myjanaza.qabr · SDK Expo 52

Pourquoi cette étape ? Android utilise FCM (Firebase Cloud Messaging) pour recevoir les notifications push. Sans le fichier google-services.json, l'app affiche l'erreur "expo-notifications: Android Push notifications…" au démarrage. Ce fichier n'est utilisé que pour les builds — pas pour le fonctionnement de l'app en elle-même.

Étape 1 — Créer le projet Firebase

1
Ouvrir la console Firebase
Va sur console.firebase.google.com et connecte-toi avec ton compte Google.
2
Créer un nouveau projet
Clique sur "Créer un projet Firebase".
Nom du projet : salat-janaza (ou ce que tu veux).
Note : les projets existants (janazaDB, etc.) ont un numéro de projet différent — il vaut mieux en créer un nouveau dédié.
3
Google Analytics Optionnel
Tu peux désactiver Google Analytics — pas utile pour les push notifications. Clique "Créer le projet" et attends quelques secondes.

Étape 2 — Ajouter l'application Android

4
Ajouter une application Android
Une fois le projet créé, clique sur l'icône Android (⚙ → "Ajouter une application") ou sur le bouton Android sur la page d'accueil du projet.

Remplis uniquement le nom du package Android : fr.myjanaza.qabr Le surnom et la clé SHA-1 sont optionnels — laisse vide et clique "Enregistrer l'application".
5
Télécharger google-services.json Important
Clique sur "Télécharger google-services.json".
Clique "Suivant", "Suivant", "Continuer dans la console" — on n'a pas besoin d'ajouter le SDK Firebase dans le code.

Étape 3 — Placer le fichier dans le projet

6
Copier le fichier à la racine du projet
Place le fichier téléchargé ici (à côté de app.json) :
qabr_mobile/
  google-services.json  ← ici
  app.json
  package.json
  index.js
  src/
Ce fichier ne doit pas être dans src/ ou android/ — uniquement à la racine.
7
Vérifier app.json
La ligne suivante a déjà été ajoutée automatiquement dans app.json → section android : "googleServicesFile": "./google-services.json" Rien à faire de ton côté ici.

Étape 4 — Rebuild l'application

8
Lancer le build EAS Android
Dans le terminal, à la racine de qabr_mobile/ : eas build --platform android --profile development Ce build inclura le google-services.json et configurera FCM automatiquement. Le résultat est un .apk à installer directement sur le téléphone.
9
Tester les notifications
Installe le .apk sur Android, connecte-toi, déclare une janaza dans une mosquée à laquelle tu es abonné → tu dois recevoir la notification push.
L'erreur "expo-notifications: Android Push notifications…" doit avoir disparu.

Résumé des fichiers et liens

  • Console Firebase : console.firebase.google.com
  • Package Android à utiliser : fr.myjanaza.qabr
  • Fichier à télécharger : google-services.json
  • Emplacement du fichier : qabr_mobile/google-services.json (racine)
  • Ligne dans app.json (déjà ajoutée) : "googleServicesFile": "./google-services.json"
  • Commande rebuild : eas build --platform android --profile development
